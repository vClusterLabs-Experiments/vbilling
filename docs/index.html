How it works

Three steps to tenant billing

vBilling handles metrics collection and event delivery. Lago handles pricing, plans, and invoicing. You set the prices.

01 — Discover

Discover

Automatically finds all virtual clusters via StatefulSet labels or the vCluster Platform API. No manual registration needed.

02 — Meter

Meter

Collects 9 resource metrics every 60 seconds: CPU, memory, GPU (type-aware), storage, network egress, and more.

03 — Bill

Bill

Sends usage events to Lago for aggregation, plan-based pricing, and automatic invoice generation each billing period.

Features

Everything you need to bill tenants

Whether you run a shared platform or dedicated GPU clusters, vBilling adapts to your deployment model automatically.

hive

Shared nodes

Bill by actual pod-level CPU and memory consumption when multiple tenants share cluster nodes.

admin_panel_settings

Private nodes

Bill full dedicated node capacity when tenants own bare-metal infrastructure with GPUs or high-memory.

memory_alt

GPU type-aware

Detects GPU models (H100, A100, T4) from node labels. Price each type differently in Lago.

data_usage

DCGM integration

Query NVIDIA DCGM via Prometheus for real GPU utilization metrics, not just allocation counts.

chart_data

Network egress

Meter outbound network traffic per namespace through Prometheus for bandwidth-based billing.

desktop_cloud

Spot / on-demand

Detect spot instance nodes and apply configurable discounts for accurate cost attribution.

tune

LoadBalancer tracking

Track LoadBalancer service hours per virtual cluster for infrastructure-level billing.

door_open

Customer portal

Lago provides a built-in customer portal where tenants view usage, invoices, and prepaid wallets.

autorenew

Auto-discovery

Watches for vCluster StatefulSets and Platform API resources. Customers created in Lago automatically.

Quick start

Up and running in 3 steps

Deploy Lago, install vBilling, and configure your pricing. Usage events start flowing immediately.

1

Deploy Lago

Spin up the open-source billing engine with Docker Compose. UI at :8080, API at :3000.

# Clone vBilling and start Lago git clone https://github.com/vClusterLabs-Experiments/vbilling.git cd vbilling/deploy/lago openssl genrsa 2048 > lago_rsa.key echo "LAGO_RSA_PRIVATE_KEY=$(base64 -i lago_rsa.key | tr -d '\n')" > .env docker compose --env-file .env up -d
2

Install vBilling

Deploy the controller via Helm. Point it at your Lago instance with your API key.

helm upgrade --install vbilling deploy/helm/vbilling \ --namespace vbilling-system --create-namespace \ --set lago.apiURL=http://lago-api:3000 \ --set lago.apiKey=YOUR_LAGO_API_KEY
3

Configure pricing

vBilling creates a skeleton plan with $0 pricing. Set your rates in the Lago UI or API.

# In Lago UI → Plans → vCluster Standard # Set prices per metric: CPU Core-Hours $0.065 Memory GB-Hours $0.009 GPU Hours (H100) $4.50 Storage GB-Hours $0.0002 Network Egress GB $0.09 Private Node Hours $25.00 LoadBalancer Hours $0.025

For neoclouds

Built for GPU cloud providers

If you sell GPU compute on Kubernetes, vBilling gives you the billing layer you need — without building it yourself.


Neocloud providers give each customer a virtual cluster with private, dedicated GPU nodes. vBilling detects the dedicated hardware, meters full node capacity, and feeds usage into Lago for monthly invoicing.

  • Detects dedicated H100, A100, L40S nodes per tenant
  • Bills full node capacity — CPU, memory, and GPUs
  • GPU type-specific pricing via Lago plans
  • DCGM integration for real utilization tracking
  • Lago handles invoices, wallets, and Stripe integration
