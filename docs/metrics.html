Metrics Reference

vBilling tracks 9 billable metrics across CPU, memory, storage, GPU, network, and infrastructure. Each metric is sent to Lago as a usage event with specific properties.

i
All metrics are aggregated using Lago's sum_agg aggregation type. Values are multiplied by the collection interval (in hours) to convert from instantaneous readings to hourly units.

1. CPU Core-Hours

vcluster_cpu_core_hours

Shared Mode Private Node Mode
Source
Kubernetes Metrics Server (PodMetrics API)
Unit
Core-hours
Formula
current_cpu_cores x interval_hours x cost_multiplier
Lago Field
cpu_core_hours

Measures the actual CPU usage of all pods running in the vCluster namespace. The collector queries the metrics-server PodMetrics API, sums up the CPU usage (in millicores) across all containers, and converts to core-hours by multiplying by the collection interval in hours.

In shared mode, only namespace pods are counted. In private node mode, the full CPU capacity of dedicated nodes is also added to the total.

The cost_multiplier adjusts for spot vs. on-demand pricing. If 50% of pods run on spot nodes with a 60% discount, the multiplier is 0.70.

Lago event properties

PropertyTypeDescription
cpu_core_hoursfloatBillable core-hours (with cost multiplier applied)
raw_cpu_coresfloatInstantaneous CPU usage in cores
cost_multiplierfloatSpot/on-demand cost adjustment factor
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

2. Memory GB-Hours

vcluster_memory_gb_hours

Shared Mode Private Node Mode
Source
Kubernetes Metrics Server (PodMetrics API)
Unit
GB-hours
Formula
current_memory_gb x interval_hours x cost_multiplier
Lago Field
memory_gb_hours

Measures actual memory consumption of all pods in the vCluster namespace. Memory is read from the metrics-server in bytes, converted to GB (bytes / 1024^3), then multiplied by the interval to produce GB-hours.

In private node mode, the total memory capacity of dedicated nodes is added on top of shared-namespace pod usage.

Lago event properties

PropertyTypeDescription
memory_gb_hoursfloatBillable GB-hours (with cost multiplier)
raw_memory_gbfloatInstantaneous memory usage in GB
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

3. Storage GB-Hours

vcluster_storage_gb_hours

Shared Mode
Source
PersistentVolumeClaims (PVC spec.resources.requests)
Unit
GB-hours
Formula
total_pvc_storage_gb x interval_hours
Lago Field
storage_gb_hours

Sums the requested storage size of all bound PVCs in the vCluster namespace. Only PVCs with status.phase == Bound are counted. Storage billing is not affected by spot/on-demand node cost multipliers.

Lago event properties

PropertyTypeDescription
storage_gb_hoursfloatBillable storage GB-hours
raw_storage_gbfloatTotal PVC storage in GB at time of collection
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

4. GPU Hours

vcluster_gpu_hours

Shared Mode Private Node Mode
Source
Pod nvidia.com/gpu resource requests + node labels for GPU type
Unit
GPU-hours
Formula
gpu_count x interval_hours (per GPU type)
Lago Field
gpu_hours

Counts the number of nvidia.com/gpu resources requested by running pods, grouped by GPU type. The GPU type is detected from node labels on the node where the pod is scheduled. One event is sent per distinct GPU type.

GPU type detection (label priority order)

  1. nvidia.com/gpu.product -- NVIDIA GPU Operator (e.g., "NVIDIA-A100-SXM4-80GB")
  2. nvidia.com/gpu.machine -- alternative NVIDIA label
  3. accelerator -- GKE generic accelerator label
  4. cloud.google.com/gke-accelerator -- GKE-specific (e.g., "nvidia-tesla-t4")
  5. k8s.amazonaws.com/accelerator -- EKS (e.g., "nvidia-tesla-a100")
  6. node.kubernetes.io/instance-type -- fallback: cloud instance type

In private node mode, GPU counts are read from the node's status.capacity["nvidia.com/gpu"] field, and GPU types are detected using the same label priority.

Lago event properties

PropertyTypeDescription
gpu_hoursfloatBillable GPU-hours for this GPU type
gpu_countintNumber of GPUs of this type
gpu_typestringGPU model name (normalized)
billing_modestring"private_node" when from dedicated nodes, absent in shared mode
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

5. GPU Utilization

vcluster_gpu_utilization

Shared Mode Requires Prometheus
Source
DCGM Exporter via Prometheus (DCGM_FI_DEV_GPU_UTIL)
Unit
Percentage-hours (utilization score)
Formula
avg_utilization_percent x interval_hours
Lago Field
gpu_util_score

Queries DCGM GPU utilization from Prometheus for all GPUs in the vCluster namespace. The average utilization percentage is multiplied by the interval to produce a utilization score. Requires PROMETHEUS_URL to be configured and the DCGM Exporter running in the cluster.

Also queries DCGM_FI_DEV_FB_USED and DCGM_FI_DEV_FB_FREE for GPU memory usage tracking. This metric is primarily informational and can be used for billing adjustments or overuse surcharges.

Lago event properties

PropertyTypeDescription
gpu_util_scorefloatUtilization score (avg% x hours)
avg_utilization_pctfloatAverage GPU utilization percentage (0-100)
gpu_countintNumber of GPUs with utilization data
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

6. Network Egress

vcluster_network_egress_gb

Shared Mode Requires Prometheus
Source
Prometheus container_network_transmit_bytes_total
Unit
GB
Formula
sum(rate(container_network_transmit_bytes_total{namespace}[5m])) x 300 / 1024^3
Lago Field
egress_gb

Measures outbound (transmit) network traffic for all containers in the vCluster namespace. Uses a 5-minute rate window from Prometheus to estimate bytes per second, then multiplies by 300 seconds. Requires PROMETHEUS_URL to be set.

Ingress traffic (container_network_receive_bytes_total) is also collected and tracked internally but not billed by default.

Lago event properties

PropertyTypeDescription
egress_gbfloatEgress traffic in GB
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

7. LoadBalancer Hours

vcluster_lb_hours

Shared Mode
Source
Kubernetes Services with spec.type == LoadBalancer
Unit
Hours
Formula
loadbalancer_count x interval_hours
Lago Field
lb_hours

Counts the number of Services of type LoadBalancer in the vCluster namespace. Each LoadBalancer service incurs an hourly charge. This covers the cost of cloud load balancer resources (e.g., AWS NLB/ALB, GCP LB) provisioned by the vCluster.

Lago event properties

PropertyTypeDescription
lb_hoursfloatTotal LoadBalancer service-hours
lb_countintNumber of LoadBalancer services
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

8. Instance Hours

vcluster_instance_hours

Shared Mode Private Node Mode
Source
Per-vCluster (always sent for every tracked vCluster)
Unit
Hours
Formula
interval_hours (flat 1x per vCluster)
Lago Field
instance_hours

A flat hourly charge for running a vCluster instance. This event is always sent for every tracked vCluster, regardless of resource usage. It represents the base cost of operating a vCluster -- the platform overhead, control plane resources, and management infrastructure.

Lago event properties

PropertyTypeDescription
instance_hoursfloatHours the vCluster was running
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster

9. Private Node Hours

vcluster_private_node_hours

Private Node Mode Only
Source
Node labels: vcluster.loft.sh/managed-by, vcluster.loft.sh/cluster
Unit
Node-hours
Formula
private_node_count x interval_hours
Lago Field
private_node_hours

Counts the number of dedicated/private nodes allocated to a vCluster tenant. Only sent when private nodes are detected via node labels. This represents the cost of reserving entire physical or virtual machines for a single tenant.

For full details on how private nodes are detected and billed, see the Private Nodes guide.

Lago event properties

PropertyTypeDescription
private_node_hoursfloatTotal node-hours (count x interval)
node_countintNumber of dedicated nodes
billing_modestringAlways "private_node"
vcluster_namestringName of the vCluster
vcluster_namespacestringNamespace of the vCluster
